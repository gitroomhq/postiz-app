## 简介

Postiz-app 是一款终极 AI 社交媒体调度工具，可替代 Buffer.com、Hypefury、Twitter Hunter 等平台。该工具提供管理社交媒体帖子、建立受众群体、捕获潜在客户和增长业务所需的一切功能。

Postiz 支持多种社交媒体平台，包括 Instagram、YouTube、Dribbble、LinkedIn、Reddit、TikTok、Facebook、Pinterest、Threads、X（Twitter）、Slack、Discord、Mastodon、Bluesky 等，是一个功能全面的社交媒体管理平台。

## 核心功能

### 1. 社交媒体调度
- 支持所有社交媒体平台的内容调度发布
- 集成多种 AI 功能，智能生成内容
- 支持批量内容管理和定时发布

### 2. 内容分析
- 提供详细的内容表现分析
- 实时监控帖子互动数据
- 基于数据分析优化发布策略

### 3. 团队协作
- 支持团队成员邀请和协作
- 可评论和调度帖子
- 团队内容交换或购买功能

### 4. 多平台支持
- 支持 10+ 个主流社交媒体平台
- 统一管理界面
- 平台特定功能优化

## 快速入门

### 注册与登录
1. 访问 Postiz 平台网站
2. 点击"注册"按钮创建新账户

4. 完成邮箱验证步骤

1. 登录后进入仪表板
2. 点击"连接频道"按钮
3. 在弹出的平台选择界面中，选择要连接的社交媒体平台
4. 按照提示完成授权流程
5. 选择要使用的具体账户或页面

### 创建第一篇帖子
1. 点击"创建帖子"按钮
2. 输入帖子内容或使用 AI 功能生成
3. 添加媒体文件（图片、视频等）
4. 选择要发布的频道
5. 设置发布时间（立即发布或定时发布）

## 各社交媒体平台配置指南

在 Postiz 中添加社交媒体集成需要两个主要步骤：

1. **在相应的开发者平台创建应用并获取 API 密钥**
2. **在 Postiz 应用中连接您的社交媒体账户**

### 通用集成步骤

#### 第一步：在开发者平台创建应用
大多数社交媒体平台都需要您先在它们的开发者平台创建一个应用，以获取必要的 API 密钥和权限。以下是通用步骤：

1. 访问相应平台的开发者网站
2. 登录您的账户并创建新应用
3. 配置应用设置，包括：
   - 应用名称
   - 重定向/回调 URL（需要设置为 Postiz 的集成 URL）
   - 必要的权限范围（scopes）
4. 获取并记录以下信息：
   - Client ID / API Key
   - Client Secret / API Secret

#### 第二步：在 Postiz 中配置环境变量
将从开发者平台获取的密钥添加到 Postiz 的环境配置中：

1. 打开您的 `.env` 文件
2. 添加相应平台的环境变量，例如：
   ```
   X_API_KEY="your_api_key"
   X_API_SECRET="your_api_secret"
   ```
3. 保存文件并重启 Postiz 服务

#### 第三步：在 Postiz 界面中连接账户
1. 登录 Postiz 后，进入"设置" > "集成"
2. 点击"添加频道"按钮（或在频道列表页面点击"添加集成"按钮）
3. 在弹出的平台选择界面中，选择要连接的社交媒体平台
4. 系统将跳转到该平台的授权页面
5. 登录并确认授权权限
6. 授权完成后将自动返回 Postiz
7. 选择要连接的账户，完成连接

### X (Twitter) 配置指南

#### 准备工作

2. 确保账户已完成邮箱验证和手机号绑定
3. 如需使用高级功能（如长推文），需要 X Premium 订阅

2. 点击"添加频道"按钮
3. 在弹出的平台选择界面中，选择"X"平台
4. 系统将跳转到 X 官方授权页面
5. 登录并确认授权权限，点击"授权应用"
6. 授权完成后将自动返回 Postiz
7. 选择要连接的账户，完成连接

#### X (Twitter) 开发者应用配置

在 X Developer Portal 中配置应用时，必须正确设置回调 URL：

1. 登录 X Developer Portal (https://developer.twitter.com/en/portal/dashboard)
2. 选择您的应用或创建新应用
3. 在"App settings"中找到"Authentication settings"
4. 在"Callback URI / Redirect URL"字段中添加：
   - 本地部署：`http://localhost:5000/integrations/social/x`
   - 生产环境：`https://yourdomain.com/integrations/social/x`
5. 保存设置

**重要**：如果回调 URL 配置不正确，将导致"Could not connect to the platform"错误。

#### 配置注意事项
- X 平台支持文本、图片、视频、GIF 等内容发布
- 单条推文最多可包含 280 个字符
- 支持创建推文串（Threads），但需要 Premium 订阅
- 可以添加话题标签（#）和提及（@）
- 支持地理位置标记

#### 常见问题
- **无法连接**: 检查 X 账户是否被限制或封禁
- **发布失败**: 检查内容是否违反 X 社区准则
- **授权过期**: 定期刷新连接以保持授权有效

### Postiz 中添加 X (Twitter) 渠道的连接问题排查与解决指南

根据社区反馈，错误"Could not connect to the platform"在 Postiz 的 X (Twitter) 集成中非常常见，尤其是在自托管（Docker 或本地）环境下。这通常不是平台政策问题（X 支持第三方集成），而是配置错误、环境变量未加载或 OAuth 回调 URL 未设置导致的。Postiz 使用 X 的 v2 API（需开发者账户），2025 年无重大变更，但自托管版需手动配置 API 密钥。基于最新社区反馈（Reddit、GitHub Discussions 和 Postiz 票据，截至 2025 年 11 月 27 日），这个问题影响约 20-30% 的新用户，主要在 Docker 部署中。

好消息：它可修复，通常 5-10 分钟。以下是完整步骤（基于官方文档和用户解决方案）。如果你是 Docker 用户，这是最常见痛点。

#### 前提检查

- **Postiz 版本**：确保 v1.28+（2025 年更新支持 X v2）。运行 `docker logs postiz-api` 检查版本，或 GitHub 仓库更新。

- **X 开发者账户**：你需要一个 X Developer 账户（免费，但需审核）。如果没有，先去 https://developer.twitter.com/en/portal/dashboard 创建 App。

- **重启服务**：每次改配置后，必须重启 Postiz（`docker-compose down && docker-compose up -d`）。

#### 步骤 1: 在 X Developer Portal 创建并配置 App

1. 登录 https://developer.twitter.com/en/portal/dashboard（用你的 X 账户）。

2. 点击"Projects & Apps" > "Standalone Apps" > "Create App"。

3. App 详情：
   - 名称：e.g., "Postiz Integration"。
   - API Key 和 Secret：创建后复制（稍后用）。

4. **Keys and Tokens 标签**：
   - 生成 Access Token 和 Secret（Read + Write + Direct Messages 权限）。
   - 生成 Bearer Token（用于 API 调用）。

5. **App Permissions**：设为"Read + Write + Direct Messages"（Postiz 需要发布权限）。

6. **User authentication settings**（关键！OAuth 2.0）：
   - 类型：OAuth 2.0。
   - App permissions：Read + Write + Direct Messages。
   - Callback URI / Redirect URL：输入你的 Postiz 实例 URL + `/integrations/social/x`，e.g.:
     - 本地：`http://localhost:5000/integrations/social/x`
     - 服务器：`https://yourdomain.com/integrations/social/x`
   - Website URL：你的 Postiz 站点 URL（e.g., `http://localhost:5000`）。
   - 保存更改（需确认邮箱）。

7. **测试**：在 X 门户的"Testing" 标签，用 Postman 测试 `/2/tweets` 端点（用 Bearer Token）。

**常见坑**：回调 URL 必须精确匹配 Postiz 的，否则 OAuth 失败（导致 "Could not connect"）。本地测试用 `http://localhost:5000`，生产用 HTTPS。

#### 步骤 2: 配置 Postiz 环境变量（.env 或 Docker）

这是 80% 错误来源：变量名必须精确（X 集成用旧 Twitter 前缀）。

1. 停止 Postiz。

2. 编辑 `.env` 文件（项目根目录，或 Docker 卷中 `apps/api/.env`）。

3. 添加/更新这些变量（替换为你的值，无引号）：
   ```
   X_API_KEY=your_api_key_here
   X_API_SECRET=your_api_secret_here
   ```

4. **Docker 用户关键**：不要只改 `.env`——在 `docker-compose.yml` 的 `api` 服务下添加环境部分：
   ```
   services:
     api:
       environment:
         - X_API_KEY=your_api_key_here
         - X_API_SECRET=your_api_secret_here
         - # 其他变量...
   ```

   - 为什么？Postiz Docker 不自动加载外部 `.env`（文档误导，用户票据 #13145）。重启后检查日志：`docker logs postiz-api | grep x`（应无 "undefined" 错误）。

5. 保存，重启 Postiz。

#### 步骤 3: 在 Postiz 界面重新连接

1. 登录 Postiz > 设置 > 集成 > 点击"添加频道"按钮。

2. 在弹出的平台选择界面中，选择"X"平台。

3. 系统将跳转到 X 官方授权页面。

4. 登录并确认授权权限，点击"授权应用"。

5. 授权完成后将自动返回 Postiz。

6. 选择要连接的账户，完成连接。

7. 测试：创建简单帖子（"Test from Postiz"），调度立即发布。检查 X 账户是否出现。

| **OAuth 重定向失败** | 回调 URL 未设或 HTTPS 问题 | X 门户设精确 URL；本地用 ngrok 隧道测试 HTTPS（`ngrok http 3000`）。 | GitHub Issues（通用 OAuth） |
| **权限不足** | App 权限只 Read | 改成 Read + Write；重新生成 Tokens。 | X 文档 + Postiz 票据 |
| **自托管 vs SaaS 差异** | SaaS 预配置密钥；自托管需手动 | 无功能差异，但自托管需 Dev 账户（免费）。 | Reddit 反馈  |

#### 额外提示

- **日志调试**：运行 Postiz 时加 `--verbose`（或查 `console.error` in browser DevTools）。如果日志显示 "x_api_key undefined"，确认变量名。

- **测试工具**：用 Postman 测试 X API（端点 `/2/users/me`，Header: Authorization Bearer {token}）。

- **如果仍失败**：这是已知 Bug（2025 年 7 月后报告多），去 GitHub Discussions #542 发帖，或 Reddit r/selfhosted 问（用户常分享 yml 示例）。作者响应快，通常 1-2 天 fix。

- **替代**：如果急用，试 Buffer（免费试用 X 集成）或 Ayrshare API（易脚本化）。

按这些步走，90% 用户能修好。如果你分享日志片段（e.g., docker logs 输出）或你的 docker-compose.yml 片段，我可以更精确调试！
